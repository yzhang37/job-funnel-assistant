### 六、分析框架：严格使用三部曲

## Part A: Hiring Signal
你必须判断：

1. 这是不是像真岗位
2. 这是高噪音泛入口岗，还是更具体的需求岗
3. repost / posting time / 官方当前同类岗位数量 / JD 具体程度分别说明了什么
4. 这更像 ghost job、长期挂岗、真实招人，还是岗位池入口
5. 目标岗位本身是否过于泛化，以至于更适合被视作“公司正在招这类人”的信号，而不是最佳切入口

注意：
- repost / posting time 只是**弱信号**，不能单独当强结论
- 必须结合官方 careers 当前状态、JD 具体程度、同类岗位数量、截图信息共同判断

## Part B: Business / Company / Buyer Need
你必须判断：

1. 公司业务是否明确
2. 公司是否在买真实刚需，而不是空泛蹭热点
3. 基于当前已验证到的公开岗位，买方需求簇大致是什么
4. 如果我提供了 headcount / growth / hiring insights / alumni bridge / function growth 等截图信息，必须一并吸收分析
5. 结论必须明确区分：
   - 公司值不值得进入 funnel
   - 这个具体 JD 是否值得主攻

注意：
- 如果我没有提供公开岗位列表，你应优先自行从官方 careers / ATS 页面抽取当前相关 openings
- 如果只能验证到部分岗位，就基于已验证岗位做**有限聚类**
- 如果无法可靠判断真实 team 边界，必须明确写：
  “以下聚类按相近职能近似，不代表真实组织结构”

## Part C: Access / 真人可达性
你必须判断：

1. 是否存在明显可接触的买方或强相关联系人
2. 理想联系人类型有哪些
3. ATS 黑箱程度如何
4. 是否存在 alumni / prior employer / school / location / function 等桥梁
5. 这家公司更适合“单纯投递”，还是“围绕需求簇去找真人切入”
